# 基于智能体协作的图书个性化推荐系统 - 项目实施精简版
## 一、核心任务清单
### （一）数据相关任务
1. 选择并获取图书数据集（优先 Goodreads、Amazon Books），包含完整图书元数据（标题、作者、出版社等）和用户交互数据（评分、评论、浏览记录）。
2. 数据预处理：处理缺失值/异常数据，统一格式字段；清洗用户评论文本（分词、去噪）；标准化图书元数据。
3. 构建图书知识图谱：梳理作者-书籍、出版社-书籍、学科分类等结构化关系。
4. 数据集划分：按 80%-10%-10% 拆分训练集、验证集、测试集。

### （二）智能体设计与实现任务
1. **用户画像分析智能体**
   - 输入：用户历史交互数据、基本信息
   - 核心功能：提取显式偏好（图书类别/主题分布）、隐性偏好（评论关键词/情感），生成结构化 user profile 向量
   - 技术实现：基于 BERT 的文本理解（关键词提取、情感分析）、评分统计特征计算、多维特征融合归一化

2. **图书内容分析智能体**
   - 输入：图书元数据、用户评论聚合结果
   - 核心功能：生成图书内容向量、补充知识图谱上下文信息、提取多维特征标签（主题/风格/难度等）
   - 技术实现：Sentence-BERT 语义向量化、知识图谱 RAG 检索、NLP 特征标签提取

3. **推荐决策智能体**
   - 输入：用户 profile、候选图书特征
   - 核心功能：四维度融合评分（协同过滤+内容相似度+知识增强+多样性）、排序推荐、生成自然语言解释
   - 技术实现：集成 SVD/矩阵分解（协同过滤）、语义匹配计算、多因素评分函数设计、LLM 解释生成

### （三）系统架构与集成任务
1. 基于 ACPs 协议搭建多智能体架构：实现协调器、注册中心、通信层、工作流引擎核心组件。
2. 标准化智能体交互：遵循 AIC/ATR/ADP/ACS 规范，定义统一消息格式、异步通信机制。
3. 端到端流程开发：用户请求 → 并行调用双分析智能体 → 候选图书检索过滤 → 推荐决策 → 输出结果+解释。
4. 多场景适配：冷启动（流行度/内容相似性策略）、热推荐（多维数据精准匹配）、探索性推荐（提升新颖性）。

### （四）测试与优化任务
1. 单体测试：验证各智能体输入输出正确性、性能指标。
2. 集成测试：验证智能体协作可靠性、通信稳定性。
3. 系统评估：基于 Precision@k、Recall@k、NDCG@k、多样性、新颖性等指标开展评估。
4. 消融研究：分析各模块（协同过滤/知识增强等）对性能的贡献度。
5. 对标测试：与传统混合推荐、MACRec/ARAG 等多智能体方法对比。
6. 搭建演示系统：完成可展示的原型版本。
