## 第 1 章 绪论

### 1.1 研究背景

随着互联网技术的快速发展，信息过载问题日益严重。推荐系统作为解决信息过载的有效手段，在电商、社交、内容平台等领域得到广泛应用。

### 1.2 研究意义

传统的推荐系统主要基于单一模型或算法，难以应对复杂的用户需求和多变的场景。多 Agent 协作系统通过分工合作，能够更好地处理复杂任务。

### 1.3 研究目标

本研究旨在设计并实现一个基于多 Agent 协作的个性化推荐系统，通过 Leader-Partner 架构实现智能化的书籍推荐。

## 第 3 章 系统设计

### 3.1 系统架构

ACPs 系统采用 Leader-Partner 架构：

```
┌─────────────────────────────────┐
│   ReadingConcierge (Leader)     │
│   - 任务编排                    │
│   - 结果整合                    │
│   - 用户交互                    │
└──────────────┬──────────────────┘
               │
    ┌──────────┼──────────┐
    │          │          │
    ▼          ▼          ▼
┌────────┐ ┌────────┐ ┌────────┐
│Reader  │ │Book    │ │Rec     │
│Profile │ │Content │ │Ranking │
└────────┘ └────────┘ └────────┘
```

### 3.2 ACPS 协议

基于 JSON-RPC 2.0 实现 Agent 间通信协议。

### 3.3 安全机制

支持 mTLS 双向认证，确保通信安全。

## 第 4 章 系统实现

### 4.1 技术栈

- **后端**: Python 3.10, FastAPI
- **协议**: JSON-RPC 2.0
- **安全**: mTLS
- **部署**: Docker

### 4.2 核心模块

#### 4.2.1 ReadingConcierge

Leader Agent，负责任务编排。

#### 4.2.2 ReaderProfile

Partner Agent，负责读者画像分析。

#### 4.2.3 BookContent

Partner Agent，负责书籍内容分析。

#### 4.2.4 RecRanking

### 5.3 评估指标

- 平均延迟

- 向量维度
- 语义质量

### 5.4 嵌入模型对比实验

**实验结果**：

| 指标 | hash-fallback | qwen3-vl-embedding | 提升 |
|------|---------------|--------------------|------|
| 平均延迟 | 1.511s | 0.233s | -84.6% |
| 向量维度 | 12 | 2560 | +21233% |
| API 成功率 | 100% | 100% | - |

**结论**：qwen3-vl-embedding 在各方面都优于 hash-fallback。
