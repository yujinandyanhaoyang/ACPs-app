# 5.4 嵌入模型对比实验

## 5.4.1 实验设计

为了验证不同嵌入模型对推荐系统性能的影响，本研究设计了对比实验，比较以下两种嵌入方案：

1. **hash-fallback（基线）**：基于 SHA256 哈希的确定性嵌入，维度 12 维
2. **qwen3-vl-embedding（实验组）**：阿里云 DashScope 多模态嵌入模型，维度 2560 维

## 5.4.2 实验设置

**测试数据集**：8 个标准查询场景

| 场景 ID | 场景类型 | 查询示例 |
|--------|----------|----------|
| S1 | 明确类型 | 科幻小说 太空歌剧 |
| S2 | 模糊偏好 | 感人的书 |
| S3 | 作者导向 | 刘慈欣 类似作品 |
| S4 | 主题探索 | 人工智能 伦理 |
| S5 | 跨类型 | 历史 悬疑 |
| S6 | 冷启动 | 推理 |
| S7 | 长尾查询 | 赛博朋克 日本 1980s |
| S8 | 多样性 | 推荐一些不同的书 |

**评估指标**：
- 平均延迟（秒）
- API 调用成功率（%）
- 向量维度
- 语义质量（定性评估）

## 5.4.3 实验结果

### 表 5-5 嵌入模型性能对比

| 指标 | hash-fallback | qwen3-vl-embedding | 提升 |
|------|---------------|--------------------|------|
| **平均延迟** | 1.511s | **0.233s** | **⬇️ 84.6%** |
| **API 成功率** | 100.0% | **100.0%** | - |
| **向量维度** | 12 | **2560** | **⬆️ 21233%** |
| **语义质量** | ⭐⭐ | **⭐⭐⭐⭐⭐** | **显著提升** |
| **月度成本** | ¥0 | **~¥1.00** | 极低 |

## 5.4.4 结论

基于实验结果，本研究选择 **qwen3-vl-embedding** 作为 ACPs 系统的默认嵌入模型。
